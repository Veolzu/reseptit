## Dangerous default value
```
db.py:10:0: W0102: Dangerous default value [] as argument (dangerous-default-value)
db.py:18:0: W0102: Dangerous default value [] as argument (dangerous-default-value)
```
These complaints are about the following kind of function:
```
def execute(sql, params=[]):
    con = get_connection()
    result = con.execute(sql, params)
    con.commit()
    g.last_insert_id = result.lastrowid
    con.close()
```
There could be a problem since the same list is shared by all the function calls and therefore one change in the list by one singular call would ripple to the every other function call after it. However in practice the code doesn't alter the elements in the list, so this wouldn't happen.
